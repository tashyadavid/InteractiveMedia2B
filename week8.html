LED’S Controlled by a Remote

We also tried building a project that used an IR sensor and a remote to control a row of LEDs. The idea was that pressing different buttons would light up different LEDs, which sounded fun and interactive. Even though it didn’t end up working during class, it gave me a basic understanding of how infrared communication works, like how remote controls send signals and how those get picked up by sensors. It was a bit tricky to troubleshoot, and we didn’t get the chance to finish it properly, but it showed me how many little parts need to work together for something to run smoothly.

Capacitive Touch Sensors

We worked with capacitive touch sensors, which detect when your skin touches or comes close to a conductive material. They basically measure changes in electrical charge, so when you touch something like a grape, it triggers a response — in our case, it played a sound through a buzzer. We tested it with foil, leaves, and a grape. Only the grape worked, but that made it even more fun. It was so cool to see it actually make a noise when you touched it, and we even managed to play a basic song by tapping different inputs. It was a really fun and unexpected way to learn about interactivity, and it made me realise how physical computing doesn’t have to feel technical — it can be playful and weird and creative.

Code Snippet:


            #include 

                CapacitiveSensor capSensor = CapacitiveSensor(4, 2);
                
                const int threshold = 1100;
                const int ledPin = 12;
                
                void setup() {
                  Serial.begin(9600);
                  pinMode(ledPin, OUTPUT);
                }
                
                void loop() {
                  long sensorValue = capSensor.capacitiveSensor(30);
                  Serial.println(sensorValue);
                
                  if (sensorValue > threshold) {
                    digitalWrite(ledPin, HIGH);
                  } else {
                    digitalWrite(ledPin, LOW);
                  }
                
                  delay(10);
                }

Arduino and a potentiometer to control an LED light

The potentiometer let us change the brightness in real time, kind of like a dimmer switch. It was interesting to see how turning a knob could be translated into both code and electrical output. It made it really clear how analog input can control output in a physical way. This helped me understand how sensors and inputs work with hardware and software to create interactive experiences.

Code Snippet:


            int ledPin = 11;
int ledBrightness = 0;

int potPin = A0;
int potVal = 0;

void setup() {
  pinMode(ledPin, OUTPUT);
}

void loop() {
  potVal = analogRead(potPin);
  ledBrightness = potVal / 4;
  analogWrite(ledPin, ledBrightness);
  delay(10);
}

Capacitive Sensor with a Buzzer

We worked with a capacitive sensor and a buzzer, where the buzzer would go off if someone was in close proximity to it. It was really interesting to see it react in real time, and it made me realise how much everyday technology works in a similar way. I also started to see how physical computing could be used creatively. Even something as simple as a buzzer could feel emotional or symbolic, depending on the context. It was fun to build, and even when it didn’t work at first, the hands-on trial and error helped everything make more sense.

Code Snippet:


            int const trigPin = 11;
int const echoPin = 12;
int const buzzPin = 2;

void setup()
{
  pinMode(trigPin, OUTPUT);
  pinMode(echoPin, INPUT);
  pinMode(buzzPin, OUTPUT);
}

void loop()
{
  int duration, distance;

  digitalWrite(trigPin, HIGH);
  delay(1);
  digitalWrite(trigPin, LOW);
  
  duration = pulseIn(echoPin, HIGH);
  distance = (duration / 2) / 29.1;
  
  if (distance <= 50 && distance >= 0) {
    tone(buzzPin, 120, 500);
  } 
  else {
    digitalWrite(buzzPin, 0);
  }

  delay(60);
}
